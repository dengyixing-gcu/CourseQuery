# 课表智能助手 - 部署到 GitHub 和 Render

## 📋 完整部署流程

### 第一步：准备 GitHub 账号

1. **注册 GitHub** (如果没有)
   - 访问 https://github.com
   - 点击 "Sign up" 注册账号

2. **创建 Personal Access Token**
   - 访问 https://github.com/settings/tokens
   - 点击 "Generate new token (classic)"
   - 勾选 `repo` 权限
   - 生成并保存 Token（只显示一次！）

### 第二步：推送到 GitHub

#### 方法 A：使用部署脚本（推荐）

```bash
# 进入项目目录
cd /home/xing/课表智能体-GitHub 版

# 运行部署脚本
chmod +x deploy_to_render.sh
./deploy_to_render.sh
```

按提示输入：
- GitHub 用户名
- GitHub Token 或密码

#### 方法 B：手动推送

```bash
# 进入项目目录
cd /home/xing/课表智能体-GitHub 版

# 初始化 Git
git init
git checkout -b main

# 添加所有文件
git add .

# 提交
git commit -m "feat: 课表智能助手"

# 添加远程仓库 (替换 YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/teacher-schedule-assistant.git

# 推送 (需要 Token 或密码)
git push -u origin main
```

### 第三步：部署到 Render

1. **访问 Render**
   - 打开 https://render.com
   - 点击 "Get Started for Free"

2. **注册/登录**
   - 可以使用 GitHub 账号直接登录
   - 或邮箱注册

3. **创建 Web Service**
   - 点击 Dashboard 右上角的 "New +"
   - 选择 "Web Service"
   - 点击 "Connect a repository"

4. **连接 GitHub 仓库**
   - 找到 `teacher-schedule-assistant`
   - 点击 "Connect"

5. **配置服务**
   
   | 配置项 | 值 |
   |--------|-----|
   | Name | teacher-schedule-assistant |
   | Region | Singapore (新加坡) |
   | Branch | main |
   | Root Directory | (留空) |
   | Runtime | Python |
   | Build Command | `pip install -r requirements.txt` |
   | Start Command | `gunicorn app:app --bind 0.0.0.0:$PORT` |

6. **选择免费计划**
   - 向下滚动到 "Instance Type"
   - 选择 "Free" ($0/month)

7. **高级设置（可选）**
   - 点击 "Advanced"
   - 添加自动部署：
     - Auto-Deploy: ✅ Enabled

8. **创建服务**
   - 点击 "Create Web Service"
   - 等待构建和部署（约 3-5 分钟）

### 第四步：验证部署

1. **查看部署日志**
   - 在 Render Dashboard 点击您的服务
   - 查看 "Logs" 标签页
   - 应该看到 "Listening on port XXXX"

2. **访问应用**
   - 点击服务页面顶部的 URL
   - 格式：`https://teacher-schedule-assistant-xxxx.onrender.com`

3. **上传课表**
   - 在首页上传您的课表 Excel 文件
   - 等待上传完成

4. **测试对话**
   - 点击 "开始对话"
   - 尝试提问：
     - "查看课表"
     - "今天有什么课"
     - "张三老师的课"

## ⚙️ 配置说明

### 环境变量

在 Render 的 "Environment" 标签页添加：

| Key | Value | 说明 |
|-----|-------|------|
| `PYTHON_VERSION` | `3.11.0` | Python 版本 |
| `FLASK_ENV` | `production` | 生产环境 |

### 自动部署

启用 Auto-Deploy 后：
- 每次推送到 GitHub 的 `main` 分支
- Render 会自动重新部署
- 无需手动操作

## 🔍 故障排查

### 问题 1: 推送失败

**错误**: `Authentication failed`

**解决**:
1. 使用 Personal Access Token 代替密码
2. 确保 Token 有 `repo` 权限
3. 重新生成 Token 并更新

### 问题 2: Render 构建失败

**错误**: `No such file or directory: requirements.txt`

**解决**:
1. 确认 `requirements.txt` 在项目根目录
2. 检查 Git 是否已提交该文件
3. 重新推送代码

### 问题 3: 应用无法访问

**症状**: 502 Bad Gateway

**解决**:
1. 查看 Render 日志
2. 确认 Start Command 正确
3. 检查端口配置（使用 `$PORT` 环境变量）
4. 等待冷启动完成（首次访问需 30-50 秒）

### 问题 4: 课表上传失败

**解决**:
1. 检查 Excel 文件格式
2. 确认包含必需列（教师、课程名称、时间、地点）
3. 查看浏览器控制台错误信息

## 🚀 优化建议

### 1. 减少冷启动时间

在 `render.yaml` 中添加健康检查：
```yaml
healthCheckPath: /health
```

### 2. 保持服务活跃（不推荐）

可以使用 UptimeRobot 等工具定期访问，但可能违反 Render 服务条款。

### 3. 数据持久化

免费计划不提供持久化存储，课表数据会在重启后丢失。建议：
- 每次重启后重新上传课表
- 或升级到付费计划使用 PostgreSQL
